Enable one toggle. Keycloak shows its own register form. No Spring Boot code needed.

User clicks "Register" — goes to Keycloak's own registration form

🔧 How to enable self-registration in Keycloak

1
Open Keycloak Admin UI at http://52.65.212.193:8180
2
Go to Realm Settings → Login tab
Realm Settings → Login → User registration → ON
3
Optional extras on the same tab:
Email as username → ON
Verify email → ON
Remember me → ON
4
Set default role for new users:
Realm Settings → User Registration → Default roles → add USER
Way B — Custom Form
Your Own Registration Form → Spring Boot → Keycloak Admin API
Full control over UI. Spring Boot calls Keycloak's Admin REST API to create the user.
1
Browser / React App
User fills YOUR custom registration form
Your design, your fields — anything you need
Name : Alice Smith Email : alice@example.com Phone : +61 400 123 456 ← custom field Password : ••••••••
2
Frontend → Spring Boot
POST to your registration endpoint
POST :8080/api/auth/register Content-Type: application/json { "username": "alice", "email": "alice@example.com", "password": "secret123" }
3
Spring Boot → Keycloak Admin API
Creates user via Keycloak Admin REST API
Spring Boot must first get an admin token, then call the users API
POST :8180/admin/realms/demo-realm/users Authorization: Bearer <admin-token> { "username": "alice", "email": "alice@example.com", "enabled": true, "credentials": [{ "type": "password", "value": "secret123", "temporary": false }] }
4
Spring Boot → Keycloak Admin API
Assigns USER role to the new user
POST :8180/admin/realms/demo-realm /users/{userId}/role-mappings/realm [{ "name": "USER" }] → 204 No Content (role assigned)
5
Spring Boot → Frontend
Returns success to frontend
HTTP 201 Created { "message": "Registration successful" }
6
Frontend → Keycloak
Auto-login: fetch JWT token
POST :8180/realms/demo-realm /protocol/openid-connect/token grant_type=password username=alice password=secret123 client_id=spring-demo-client
7
Frontend → Spring Boot
Calls API with JWT — user is in
GET :8080/api/user/me Authorization: Bearer eyJhbG... → 200 OK

Way A vs Way B

Way A — Keycloak built-in Way B — Custom form
Code needed ✅ Zero ~50 lines in Spring Boot
Custom UI / branding ❌ Keycloak's UI ✅ Fully yours
Extra fields (phone, DOB) ❌ Hard to add ✅ Easy
Email verification ✅ Built-in toggle ✅ Keycloak handles it
Password rules ✅ Configure in Keycloak ✅ You control
Who stores password Keycloak Keycloak (Spring never stores it)
Time to ship ✅ Minutes Hours
Best for Internal tools, quick demos Customer-facing products
